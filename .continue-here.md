## ▶ NEXT SESSION — paced-loop redesign (USER-DIRECTED, do this first)
**Problem:** the live demo with >~9 trucks is sim-loop-bound. `driveSimulationPaced` advances ONE sim-event-tick per wall interval and ties `tickIntervalMs = 500/multiplier`; the per-tick work (append→project→catchup→**optimizer**→snapshot) then exceeds the shrinking interval → the sim becomes compute-bound, the 64× knob does nothing, and 18–27 trucks freeze the clock.

**Fix (the user's design — correct):**
1. **Decouple cadence from speed.** FIXED wall cadence (tick every ~250ms–1s). Each wall-tick advances a sim-clock cursor by `simSpeed × wallElapsed`, processes ALL pending sim-events with `occurredAt ≤ cursor` in ONE batch, broadcasts once. `SpeedController`: `tickIntervalMs` → a FIXED cadence; **`simSpeed` (sim-ms/wall-ms) becomes the lever** (already the client tween's basis — server now matches). At 64× one 1s tick consumes a 64-sim-min window. Determinism untouched (event STREAM unchanged; only delivery batching).
2. **Debounce the optimizer** (replace the coarse `optimizerEveryTicks`): run at most once per trailing quiet-window (~500ms–1s), coalescing bursts, **off the event-processing path** (non-blocking) so it never stalls event/animation flow.
3. Then set demo `FLEET_PER_SPOKE=3` (27 trucks) and verify 27 trucks glide at 64×.
