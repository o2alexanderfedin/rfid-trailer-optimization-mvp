## Key Decisions / Lessons
- **Test the runnable entrypoint, not just the factory.** `main.ts` shipped a boot crash behind 872 green tests because nothing spawned it — same "green-but-dark" theme as the v1.0 audit. The new `main-boot.int.test.ts` spawns the compiled `dist/main.js` against a testcontainer.
- **Heavy testcontainer+sim int tests are load-fragile.** At 8-wide concurrency on 8 cores they starve and trip 120–180s timeouts; they pass in isolation. Serializing is the certain fix; shared-PG is the faster long-term one.
- **Verify before "fixing."** A suspected KPI wrong-trailer/missed-unload label swap turned out to be a misread — the panel matched the API exactly. Didn't ship a wrong change.
- **Emoji preserve data:** rendered the glyph ON the colored disc (not instead of it) so volume/state coloring + the leak-tested feature discipline survive.
