<objective>
Deliver the backend for VIZ-05 (click a trailer → its load plan + why) and UI-02 (read-only audit timeline for a package OR trailer, including the captured system recommendation).

VIZ-05: `GET /api/trailers/:id/plan` returns the trailer's rear→nose load order, loading instructions (by nose/middle/rear zone), and the plan's plain-English explanation — sourced from the latest accepted plan + the Phase-2 `instructions`/`planExplanation` renderers.

UI-02: the audit timeline today is package-only (`readAuditTimeline(db, packageId)`). UI-02 requires a TRAILER timeline too AND the captured system recommendation at each decision. This plan extends the audit projection to index trailer streams and persist the recommendation/rationale on `PlanGenerated`/`PlanAccepted` rows, plus a `readTrailerAuditTimeline` helper and a trailer-history route.

Per 05-CONTEXT.md: UI-02 "may need persisting [the recommendation] in the audit projection" — this plan does exactly that, preserving the FND-04 golden-replay invariant (the reducer stays a pure function of the stored event).

Purpose: the click-through plan detail and the audit timeline with captured rationale make the optimizer's decisions legible and auditable.
Output: the plan-detail route, the extended audit reducer + schema + read helper, and the trailer-history route.
</objective>
