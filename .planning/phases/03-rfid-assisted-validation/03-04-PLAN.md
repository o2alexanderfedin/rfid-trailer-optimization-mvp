<objective>
Add the PURE detection predicates to `@mm/sensor-fusion`: `detectWrongTrailer` and `detectMissedUnload`,
operating on the two explicit layers — PLANNED/KNOWN (from scans + the Phase-2 plan/assignment) vs OBSERVED
(the confidence-scored RFID zone/trailer estimate from Plan 02's fusion). A candidate exception fires ONLY on
positive disagreement ABOVE a confidence threshold. This is where the anti-P6 keystone lives: ABSENCE of a
read returns NOTHING and NEVER marks a package missing.

Purpose: P6 (RFID-as-truth) is the cardinal Phase-3 risk. Detection must be SEPARATE from the fusion
likelihood engine (a post-filter business rule, never fed back into the Bayesian update — per the AI Mode
consult's "separate detection from tracking"). Two layers + above-threshold-disagreement-only + absence≠missing
is the structural defense, and it must be provable by pure tests with hand-built truth tables.

Output: two pure, fully-tested detection predicates emitting candidate exceptions `{ severity, recommendedAction }`,
plus the anti-P6 keystone proving absence is never an exception. The actual event emission + projection wiring
is Plan 06; this plan delivers the pure decision logic the detector calls.
</objective>

<feature>
  <name>Pure planned-vs-observed detection predicates (anti-P6)</name>
  <files>packages/sensor-fusion/src/detection.ts, packages/sensor-fusion/src/index.ts</files>
  <behavior>
    detectWrongTrailer(planned[], observed[], config):
      - For each OBSERVED package positively seen in trailer X with confidence > threshold, where the PLANNED
        assignment for that package is a DIFFERENT trailer (or the observed trailer ≠ plannedTrailerId) ⇒ emit
        ONE WrongTrailerCandidate { observedTrailerId: X, plannedTrailerId, confidence, severity, recommendedAction }.
      - Observed in the CORRECT (planned) trailer ⇒ no candidate.
      - Observed below threshold ⇒ no candidate.
      - A package with an OBSERVATION but NO planned assignment (plannedTrailerId null) ⇒ no wrong-trailer
        exception (cannot disagree with a plan that does not exist) — log-worthy, not an exception.
      - recommendedAction: e.g. `recheck_before_departure`; escalates to `block_departure` when confidence is
        high AND pre-departure (config-driven). severity = severityFor(confidence, slaImpact).

    detectMissedUnload(planned[], observed[], departedHub, config):
      - For each package whose destHub == departedHub that is STILL positively observed (confidence > threshold)
        in the trailer AFTER TrailerDeparted ⇒ emit ONE MissedUnloadCandidate { trailerId, hubId: departedHub,
        confidence, severity, recommendedAction (return / cross-dock / over-carry / transfer) }.
      - A package no longer observed (it was unloaded) ⇒ no candidate.
      - Called only post-departure (Plan 06 gates the timing); a package not destined for departedHub ⇒ none.

    BOTH predicates:
      - ABSENCE of an observation contributes NOTHING. An empty `observed` array ⇒ empty result. A package
        that simply has no read is never in the output. (anti-P6 keystone.)
      - PURE: no wall clock, no RNG; deterministic ordering of outputs (sorted by packageId).
      - Detection decisions are NEVER written back into fuse.ts — this file imports fusion types one-way only.
  </behavior>
