zone-estimate read model from Plan 05), run the pure Plan-04 detection predicates, and on positive

becomes an auditable exception event. It composes the pure pieces (Plan 04 predicates) with the persisted
OBSERVED layer (Plan 05) and the authoritative PLANNED layer (trailer-state + the Phase-2 plan). The anti-P6
guarantee must hold END-TO-END: a noisy/lossy stream produces plausible, non-flooding exceptions and NEVER a
"missing package" from absence; the false-positive KPI proves the feed stays credible.

Output: the detector function, the inline exceptions projection + false-positive KPI, and an integration test
proving plausible exceptions on a noisy sim run with absence-never-an-exception.

<!-- The detector COMPOSES existing pieces. It is the only Phase-3 write side effect (appends exception events). -->

From @mm/sensor-fusion (Plan 04): `detectWrongTrailer(planned, observed, config)`,
`detectMissedUnload(planned, observed, departedHub, config)`, `PlannedAssignment`,
`WrongTrailerCandidate`, `MissedUnloadCandidate`, `DetectionConfig`.

The PLANNED layer source (trailer-state read model, Plan-1): `TrailerState.assignedPackageIds` is the
manifest the plan/assignment put on a trailer → assemble `PlannedAssignment { packageId, plannedTrailerId,
destHubId }` (destHub from package-location / PackageCreated; plannedTrailerId = the trailer whose
assignedPackageIds contains the package).

The OBSERVED layer source (zone_estimate read model, Plan 05): the latest fused estimate per
(packageId, trailerId) — "package observed in trailerId at confidence c".

The append path (@mm/event-store): `appendToStream(db, streamId, expectedVersion, events, occurredAt)` and
`appendWithRetry(...)` (OCC-guarded). The detector appends to the package/trailer stream. WrongTrailerDetected
/ MissedUnloadDetected envelopes (from @mm/domain, Plan 01) wrap the Plan-04 candidates.

The inline applier (runner/inline.ts): register `exceptions` in APPLIERS with its own checkpoint.

TrailerDeparted event gates missed-unload: detection for a departed hub runs AFTER its TrailerDeparted.
</interfaces>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-15 | Spoofing / Tampering | absence of a read producing a "missing" exception | mitigate | detector calls only the Plan-04 predicates (driven by positive observations); the anti-P6 end-to-end integration check asserts zero exceptions from absence |
| T-03-16 | Denial of service (alert flood) | re-running detection re-appending the same exception | mitigate | stable exceptionId dedupe + above-threshold gate + false-positive-rate KPI; idempotent inline projection |
| T-03-17 | Tampering | OCC violation when detector + sim append concurrently | mitigate | appendWithRetry (expected-version OCC + reload-and-retry) — the Phase-1 concurrency contract re-verified for the new concurrent writer |
| T-03-18 | Repudiation | exception with no rationale | mitigate | every exception event carries severity (confidence × SLA impact) + recommendedAction in its payload (auditable) |
