<objective>
Extend the optimizer scope classifier and types to handle `PackageInducted`, and wire `slaDeadlineIso → TwinBlock.deadlineMin` in the twin snapshot builder so urgency-driven re-optimization is fully functional for inducted freight.

Purpose: Without this plan, the rolling optimizer silently ignores `PackageInducted` events (Pitfall 3 from RESEARCH.md) — scope returns `[]` and the optimizer never prioritizes inducted freight. `TwinBlock.deadlineMin?` is defined in types.ts, but without the twin-snapshot-builder wiring, it is always `undefined` for inducted packages (IND-03 success criterion 2 "urgency-driven re-optimization observable when tight deadlines present" is only half-delivered).

The twin-snapshot builder (twin-snapshot.ts) reads the event log and projection tables to assemble `TwinSnapshot`. Adding a helper to query `PackageInducted` events (keyed by packageId) and mapping `slaDeadlineIso → epoch-minutes` for each inducted package is the smallest additive carry: it requires no schema migration, no new table, and follows the existing `computeMilesSinceRefuel` event-log scan pattern.

Output: scope.ts and types.ts updated; twin-snapshot.ts wired; scope unit test extended; `pnpm typecheck` green.
</objective>
