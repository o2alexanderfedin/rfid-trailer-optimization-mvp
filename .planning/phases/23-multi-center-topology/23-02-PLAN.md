<objective>
Key-scope `applyHubInventory` to the hub id(s) an event actually touches (PERF-01, P1-BLOCKING),
mirroring the exact v2.1 surgery already applied to `applyPackageLocation` / `applyTrailerState` /
`applyDriverStatus`. Add a per-event projection-cost test proving row reads are independent of hub
count (10-hub vs 100-hub fold cost equal per event).

Purpose: `applyHubInventory` is the residual full-table scan (`selectFrom("hub_inventory").selectAll()`
per event). At 10 hubs it is cheap; at 100+ hubs it is O(events x hubs) again — the SAME shape as the
v2.1 freeze, with a constant that grows with the milestone's headline feature. The topology jump MUST
NOT ship without this fix or it converts a latent O(n^2) into an active one.

Output: a key-scoped `applyHubInventory` + a cost test, byte-identical to the prior fold.

This plan is INDEPENDENT of the topology work (touches a different package, `@mm/projections`), so it
runs in parallel with plan 23-01.
</objective>
