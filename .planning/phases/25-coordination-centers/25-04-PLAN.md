<objective>
Ship the five anti-oscillation / anti-deadlock guards WITH the coordinator (COORD-04) and prove anti-livelock (COORD-05). The guards: (1) hysteresis dead-band — a metric must cross a threshold AND persist ~15 sim-min before a new suggestion; (2) seeded-jitter exponential backoff — on rejection, back off that option, jitter drawn from the coordinator substream (never Math.random); (3) sim-time TTL — each suggestion expires ~6 sim-min and self-destructs if unaccepted; (4) single-owner lease per agent — a coordinator holds a ~5 sim-min lease before advising a target, so two coordinators can't target the same agent; (5) reject-path pruning — after K=3 rejections of an option the coordinator stops re-offering it, cleared on shift/zone change. Plus the anti-livelock invariant: every agent has a feasible no-op default so each tick always closes; an all-reject agent still closes; events-per-tick stays bounded (no Zeno). All constants are NAMED + sim-time/seeded/pure (determinism-safe).

Purpose: decentralizing into N coordinators re-opens oscillation/livelock/conflict N times (Pitfalls 10 + 11). These guards are the network-stability contract — the CONTEXT calls the convergence + bounded-events tests the highest-value, first-class guards and mandates an adversarial verify. The guards must be deterministic so the demo never flaps and replay stays byte-identical.
Output: a pure `guards.ts` + a named `constants.ts`, the guard integration into stepCoordinators, and a `coordinator-stability.unit.test.ts` proving convergence + bounded events. (The lease/pruning STATE is added to the engine here; SERIALIZING it for continuation-equivalence is Plan 05.)
</objective>

The guard envelope (CONTEXT decisions + DESIGN-CONSULT Q2 — sim-time/seeded/pure):
1. Hysteresis dead-band: a metric (e.g. next-hub congestion) must cross a threshold AND persist ~15 sim-min. Pure: threshold + dwell-duration over the frozen observation history. State: per-(center, agent) "metric-above-since" sim-time marker.
2. Seeded-jitter exponential backoff: backoff = pure(rejectionCount → delay); jitter from deriveCoordinatorRng (the Plan-02 substream, distinct salt — NEVER Math.random, DET-03). State: per-(center, agent, option) backoff-until sim-time.
3. Sim-time TTL (~6 sim-min = SUGGESTION_TTL_SIM_MS): the ActionSuggested.ttlSimMs stamped in Plan 02; an unaccepted suggestion self-destructs after TTL (it is a within-tick handshake, so TTL primarily bounds a pending suggestion that survives to a later pass — drop it). Pure sim-time expiry during fold.
4. Single-owner lease (~5 sim-min = LEASE_SIM_MS): a coordinator must hold a lease { coordinatorId, expiresAtSimMs } on a target before advising it; another center sees the lease and skips. Since each spoke has ONE owning center (centerOf — Pitfall 11 single-owner partition), the lease primarily guards a boundary truck in transit between centers. State: per-agent leaseBy map.
5. Reject-path pruning (REJECT_COOLDOWN_K = 3): once an agent rejects a specific option K times the coordinator stops re-offering it; cleared on shift/zone change (a center change for the agent, or a fixed sim-time window). State: per-(center, agent, option) rejection count + cooldown.

The integration point: engine.ts stepCoordinators (Plan 02) — apply the guards as a FILTER between decideCoordinatorSuggestions (candidate generation) and emit. A candidate that fails any guard is suppressed (not emitted). The reject-pruning + backoff counts are updated when a SuggestionRejected is observed (the handshake, Plan 03) — read the agent's reject from the prior pass / the same-tick handshake result.

The anti-livelock substrate (already partly built):
- The hold suggestion is ALWAYS accepted (Plan 03 arbitrateSuggestion) — the feasible no-op default. The hub Decide default is hold (24-02); the truck default is proceed/hold. So every agent tick already closes.
- After K rejections the coordinator must STOP re-suggesting that option (guard 5) — this is what bounds events-per-tick under an all-reject scenario (Pitfall 10).

Determinism: all guard state is sim-time / seeded / pure (DESIGN-CONSULT maps each guard to determinism-safe state). The state lives in in-engine maps read each pass (like odometerByTrailer). Plan 05 serializes the lease + pruning state into SerializedWorldState for continuation-equivalence.
