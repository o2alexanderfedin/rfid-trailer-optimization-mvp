3. Parse CLI args:
   const seed = parseInt(process.argv[2] ?? "42", 10);
   const durationTicks = parseInt(process.argv[3] ?? "10000", 10);
   const opts = { seed, durationTicks };

4. WORKER MODE (child process self-invocation):
   if (process.env["CAPTURE_WORKER"] === "1") {
     const hash = createHash("sha256").update(JSON.stringify(simulate(opts))).digest("hex");
     process.stdout.write(hash + "\n");
     process.exit(0);
   }

5. PARENT MODE — 4-way reproducibility check:
   a. in-process run 1: const h1 = sha(simulate(opts))
   b. in-process run 2: const h2 = sha(simulate(opts))
   c. child process 1: spawnSync(process.execPath, [scriptPath, ...args], { env: { ...process.env, CAPTURE_WORKER: "1" } })
      parse stdout as h3
   d. child process 2: same → h4
   e. Assert all 4 equal: if any differ, print error + which run differed and exit 1
   f. If all match: print "CONFIRMED SHA-256: <hash>" and exit 0

6. Self-test (optional ~3 LOC): if seed === 42 && durationTicks === 10000, assert h1 === FLAGS_OFF_GOLDEN_SHA256 and print a self-test line ("self-test: matches FLAGS_OFF golden — capture tooling is wired correctly"). If flags are non-default (the user is capturing a new model with different opts), skip the self-test.

7. Helper:
   function sha(stream: ReturnType<typeof simulate>): string {
     return createHash("sha256").update(JSON.stringify(stream)).digest("hex");
   }

Use __filename / fileURLToPath(import.meta.url) to get the script's own path for self-invocation (tsx scripts use ESM; match existing scripts' pattern).

Do NOT add new package.json dependencies. tsx, node:crypto, node:child_process are all already available. No new devDependencies needed.
