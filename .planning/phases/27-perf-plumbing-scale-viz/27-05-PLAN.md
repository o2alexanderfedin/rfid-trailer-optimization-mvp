<objective>
PERF-03 (seams half): wire the vendored `@alexanderfedin/async-queue` (linked + proven order-preserving in 27-02)
into the three RUNTIME PLUMBING seams — bounded, O(1), FIFO — replacing the unbounded / drop-based behaviors:

- (a) worker↔optimizer handoff: bound the currently-unbounded `pending` Map / `postMessage` with a small
  `AsyncQueue<WorkerRequest>` that backpressures the live-loop instead of accumulating in-flight epochs.
- (b) DB write-batching: coalesce the per-event awaited INSERT loop into multi-row `.values([...])` commits behind
  a bounded write queue (preserving the per-stream version CAS + global-order lock inside the same transaction).
- (c) ws backpressure: replace the drop-based 256 KB skip with a per-client bounded `AsyncQueue<string>` consumer
  that dequeues and awaits `socket.send` drain → true bounded-memory backpressure.

Purpose: under a sustained continental run, unbounded in-flight epochs / per-event round-trips / dropped frames are
the runtime stall modes. Bounding all three behind FIFO queues keeps memory bounded without reordering the event
stream. This is the PERF-04-enabling plumbing; it touches NO `simulate()` path (no golden change).

Output: three bounded FIFO seams + a ws-backpressure unit test, all order-preserving (the 27-02 order test +
the in-transaction append-order are the guarantees).
