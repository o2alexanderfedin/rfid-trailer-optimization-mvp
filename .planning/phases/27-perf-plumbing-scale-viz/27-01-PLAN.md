<objective>
PERF-02: replace the two per-epoch full-event-log scans in `buildTwinSnapshot`
(`computeMilesSinceRefuel` + `buildInductionDeadlines`, each a `readAll(es, 0n)` fold) with bounded
reads of two NEW incremental projections — `trailer_fuel(trailer_id, miles_since_refuel)` and
`induction_deadline(package_id, deadline_min)` — folded by the existing key-scoped `applyInline` cursor.

Purpose: optimizer epoch latency must stop growing with run length so a sustained continental run does
not stall (the PERF-04 failure mode). This is a READ-SIDE change ONLY — `simulate()` (the determinism
golden generator) has zero dependency on `buildTwinSnapshot`, so NO golden changes.

Output: two persisted projections + appliers + DDL, twin-snapshot reading them bounded, and
rebuild-equivalence + cost-invariance tests proving the incremental fold is byte-identical to a
full-scan fold.
</objective>
