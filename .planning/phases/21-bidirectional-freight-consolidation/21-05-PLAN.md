Output: optimizer_idempotency in both schema files + the Kysely interface; scope.ts both-direction scoping; twin-snapshot ORDER BY; rolling-service durable claim + PlanSuperseded co-commit; a BOUNDED int test (1 epoch + 1 restart).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Optimizer shell → Postgres | The rolling service is the ONLY side-effecting writer; the threat surface is an idempotency race / double-claim and a non-deterministic scopeHash, plus the (low) SQL-injection surface on the new table |
| Projection reads → scopeHash | Unordered SQL feeding the canonical hash is a silent integrity bug (a freeze-window shift masquerade) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-13 | Tampering | Idempotency race / double-claim on optimizer_idempotency | mitigate | UNIQUE(horizon_start,horizon_end,scope_hash) + INSERT…ON CONFLICT DO NOTHING RETURNING makes the epoch claim atomic at the DB; a duplicate claim RETURNs 0 rows and skips the append; the status column (PROCESSING/COMPLETED/FAILED) enables crash-mid-epoch recovery. The int test asserts the 0-row duplicate + restart survival. |
| T-21-14 | Tampering | Non-deterministic scopeHash via unordered SQL | mitigate | Explicit .orderBy("hub_id"/"trailer_id"/"driver_id") on the twin-snapshot reads makes the array order feeding canonicalize/scopeHash stable across restarts when both directions populate hub_inventory; the int test asserts scopeHash stability across the simulated restart. |

| T-21-16 | Tampering | SQL injection on the new table/queries | mitigate | All queries use parameterized Kysely (insertInto/.values/.onConflict/.where) — no string concatenation of values into SQL. Low severity (no user-supplied input reaches the optimizer), noted per the security threat model. |

No high-severity (HIGH) threats remain — the idempotency race and scopeHash determinism are mitigated by DB-level atomicity + ordered reads, both witnessed by the bounded int test. ASVS L1: parameterized queries (V5) satisfied.
