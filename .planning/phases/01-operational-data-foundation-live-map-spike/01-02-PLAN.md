## Phase Goal

**As a** logistics operator, **I want to** every domain change recorded as a typed, validated, append-only event, **so that** the operational twin is auditable and replayable.

<objective>
Define `packages/domain`: the zero-dependency closed set of Phase-1 domain entities and the `DomainEvent` discriminated union (with a `schemaVersion` discriminator), plus the zod-validated typed ingestion boundary. This is the single contract every other package crosses (FND-01 typed events, FND-03 typed ingestion). It bakes in P11 (event schema versioning) before many event types exist.

Purpose: A closed, validated event contract prevents the "untyped JSONB drift" debt (PITFALLS P11) and gives the event store, projections, simulation, and API one shared, type-safe vocabulary. Pure types + pure validation = trivially TDD-able and reusable by all later phases without rework.
Output: `@mm/domain` exporting entities, the `DomainEvent` union, per-event zod schemas, and `validateEvent`.
</objective>
