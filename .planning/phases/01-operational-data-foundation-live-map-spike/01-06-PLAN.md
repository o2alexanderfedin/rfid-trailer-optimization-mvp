## Phase Goal

**As a** logistics operator, **I want to** query a package's last location and full timeline, a trailer's contents, and a hub's inventory over HTTP, and stream live trailer snapshots, **so that** I can answer operational questions and feed the live map.

<objective>
Implement `packages/api` (the composition root) plus the two derived projections it needs: the catch-up audit-timeline (FND-08) and geo-track projections. Expose the FND query endpoints — package last-location+confidence+timestamp (FND-05), trailer contents (FND-06), hub inventory (FND-07), package audit timeline (FND-08) — and a ws channel pushing trailer-position/twin snapshots per sim tick (feeds the Plan 07 map).

Purpose: This turns the operational twin into answerable HTTP queries and a live stream — the "where was package X / what's on trailer T" success criteria — and provides the geo-track projection + ws snapshots the map slice consumes. The API is the only package that wires concrete infrastructure together (composition root), keeping lower packages pure.
Output: `@mm/api` Fastify server with REST query routes + ws snapshots; `@mm/projections` gains catch-up audit-timeline + geo-track.
</objective>
