## Phase Goal

**As a** logistics operator and demo audience, **I want to** a deterministic simulator that generates a seeded USA hub-network event stream, **so that** the operational twin and map have real, reproducible data to render.

<objective>
Implement `packages/simulation`: the USA hub-and-spoke network model (~10 real metro hubs with lat/long + great-circle linehaul routes — SIM-01) and a seeded, deterministic tick/event-queue engine that emits package scans, trailer trips, and arrivals as typed domain events on a virtual clock (SIM-02). The simulator is the ONLY data source for every later phase, so determinism is non-negotiable: same seed -> byte-identical event stream.

Purpose: Without the simulator there is no data; with a non-deterministic one, every replay/golden test downstream is unreliable. Building it now (right after the event store + projections contract) lets the whole stack run on real data immediately and de-risks the map.
Output: `@mm/simulation` exporting the hub network, seeded RNG, virtual clock, and `runSimulation` driving events into the event store.
</objective>
