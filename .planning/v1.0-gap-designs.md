## F-06 / OPT-02 ‚Äî wire min-cost-flow into the live epoch

**Verified gap:** `runEpoch` (packages/optimizer/src/rolling/epoch.ts) contains ZERO refs to
`assignFreight`/`minCostFlow`/`buildTimeExpandedGraph`. Live freight routing is VRPTW
(`routeTrailers`) only. MCF + the time-expanded graph are fully implemented + glpk-oracle-validated
but **dead code on the live path**. (Audit correction: the barrel DOES export `assignFreight` ‚Äî
`optimizer/src/index.ts` ‚Üí `flow/index.ts`. The false claim is the live-loop.ts:22-24 comment.)

**Approach ‚Äî MCF runs BEFORE VRPTW (assign-then-sequence), NOT instead.** `routeTrailers`
consumes per-trailer pre-assigned `Stop[]` (it sequences, it cannot assign freight‚Üílegs ‚Äî its own
docs say "VRPTW is explicitly NOT min-cost flow"). MCF answers the orthogonal OPT-02 question:
which blocks flow over which legs at min cost under shared edge/hub capacity.

**Change plan:**
1. NEW `packages/optimizer/src/flow/freight-stage.ts` ‚Äî PURE `assignFreightForEpoch(twin, epoch, cfg?, horizonMin?) => { assignments: readonly FreightAssignment[]; flowCost: number; feasible: boolean }`:
   (a) map twin.hubs/routes ‚Üí `OptimizerNetwork`; (b) `OptimizerSchedule` = one `ScheduledTrip` per in-scope trailer at its departMin; (c) `OptimizerScope` {hubIds, trailerIds, horizon=[nowMin, nowMin+(horizonMin??3*TRANSIT_MIN)], timeStepMin:15}; (d) `buildTimeExpandedGraph(network,schedule,scope,cfg??DEFAULT_GRAPH_CONFIG)`; (e) derive `Supply[]` from twin blocks: +volume at origin hub@departTimestep, ‚àívolume at nextUnloadHubId@arrivalTimestep, **Œ£ MUST = 0** ‚Äî drop any block whose origin OR sink node is absent from the graph (keep balance); (f) `assignFreight(graph,supplies)` + `minCostFlow(graph,supplies)`; return assignments + flowCost (0 if infeasible) + feasible. PURE (no clock/RNG), deterministic (id-sorted), **FAIL-SOFT** (empty/no-trip-edges ‚Üí `{assignments:[],flowCost:0,feasible:true}`).
2. `packages/optimizer/src/flow/index.ts` ‚Äî export `assignFreightForEpoch` + result type.
3. `packages/optimizer/src/rolling/types.ts` ‚Äî add OPTIONAL `freightAssignment?: {assignments:{blockId,legEdgeIds[],cost}[]; flowCost:number; feasible:boolean}` to `EpochResult` (additive, non-breaking).
4. `packages/optimizer/src/rolling/epoch.ts` (`runEpoch`) ‚Äî after `buildTwin`, `const freight = assignFreightForEpoch(twin, epoch)`; thread `freightAssignment: freight` into ALL return sites. **Do NOT change selectPlan inputs / candidate metrics / payloads** ‚Äî MCF must run + be observable WITHOUT changing the deterministic winner (preserves idempotency/feasibility/selectPlan tests).
5. `packages/api/src/optimizer/live-loop.ts` ‚Äî fix the false comment (~22-24); describe the real two-stage flow. OPTIONAL: add `freightAssignment` to `OptimizerRecommendationsDto` (routes/optimizer.ts) so MCF output is API-visible.

**TDD:** (1) RED unit in epoch.test.ts: `runEpoch(...).freightAssignment` defined, `.assignments.length>=1` with non-empty `legEdgeIds` + integer cost, `flowCost === Œ£ assignment costs`, for the routable T1 fixture. (2) RED integration (live-demo.int.test.ts or live-loop.test.ts): live path produces a feasible freightAssignment with assignments. (3) GREEN. (4) REGRESSION must stay green incl. the KEYSTONE deep-equal determinism test (two identical runEpoch ‚Üí DEEP-EQUAL incl. new field), feasibility-vs-cost, selectPlan winner, freeze-idempotency, assign-freight/min-cost-flow/glpk-oracle, live-demo (a)-(e). Build first (F-05 trap).

**Risks:** Supply imbalance ‚Üí minCostFlow THROWS (pair +/‚àí, drop off-graph nodes). Horizon must fit in-scope trips or MCF silently assigns nothing (derive horizonEnd from max departMin+travel; assert non-empty trip edges in the fixture). Determinism: id-sort all Map/Set iteration. Build freight network from the in-scope twin (anti-P9), not the full snapshot.

## F-07 / SNS-05 ‚Äî make the sim produce a missed-unload (over-carry)

**Verified gap:** every `TrailerDeparted.fromHubId`=center, every package `destHubId`=spoke,
`arriveTrailer` unloads ALL packages unconditionally, no spoke-origin departure ‚Üí the gate
`destHubId==departedHub` is unsatisfiable. Detector (`detectMissedUnload`) is correct + wired but
can never fire. Comment detection-config.ts:89-91 documents the deferral honestly.

**Approach (a) ‚Äî bounded seed-controlled over-carry in engine.ts, opt-in, off-by-default
(keeps golden byte-identical).** Use a SEPARATE seeded substream (seed XOR a salt distinct from the
RFID salt) so it never perturbs `rng`/`rfidRng`. In `arriveTrailer`, before unloading, hold back at
most ONE carried package per arrival via a draw against the rate; unload the rest. For the held-back
package emit a **spoke-origin `TrailerDeparted`** (fromHubId=spoke, toHubId=center) + (RFID on) a
portal read so it's positively observed aboard after the spoke is recorded departed; schedule a
return arrival at center to unload it. Then `departedHubs` records the spoke, `destHubId==spoke` and
is observed above threshold ‚Üí `detectMissedUnload` fires via the UNCHANGED detector.

**Change plan:**
1. `packages/simulation/src/engine.ts` (`arriveTrailer`, `generate`) ‚Äî optional `overCarry` rate in `SimulateOptions`; new seeded substream (seed XOR salt ‚âÝ RFID salt); hold back ‚â§1 carried package; emit spoke-origin `TrailerDeparted` + portal read; center return arrival to unload it. ALL randomness via the new substream, ALL time via VirtualClock.
2. `packages/api/src/detection-config.ts` ‚Äî exported demo over-carry config with calibrated rate; replace the 89-91 deferral comment with a wired note + sweep table.
3. `packages/api/src/sim/driver.ts` (`driveSimulation`, `driveSimulationPaced`) ‚Äî optional `overCarry` threaded into `simulate` only when defined. departedHubs/destHubIndex/detectorReads already carry it through the existing gate.
4. `packages/api/src/main.ts` ‚Äî pass the demo over-carry config into `driveSimulationPaced` alongside demo RFID config.

**TDD:** RED first. Simulation unit: overCarry rate 1 yields a `TrailerDeparted` with fromHubId‚âÝcenter whose packageIds include a package whose destHubId==that fromHubId; same seed+rate byte-identical; WITHOUT overCarry byte-identical to golden. Keystone API integration (real PG, mirrors live-demo): seed 4242, 120 ticks, demo RFID + demo over-carry; `GET /kpis` asserts `missedUnloadCount>0` and the feed has a `MissedUnloadDetected` (NOT a hand-fed predicate). Determinism: drive twice, identical missedUnloadCount + packageIds. Regression: determinism, rfid-determinism, FND-04 golden-replay, detector.int, live-demo (wrongTrailerCount>0), scenario all green. Build first.

**Risks:** FND-04 golden-replay must replay spoke-origin + return-leg byte-identically. If randomness draws from rng/rfidRng ‚Üí breaks golden + rfid-determinism (salted substream + off-by-default mitigates). Fusion confidence saturates ~0.33-0.45 vs threshold 0.34 ‚Üí the over-carried package needs a corroborated read (portal + dwell burst) to clear the gate ‚Äî verify the live test fires. Unload at center on return so it doesn't skew utilization/SLA. Calibrate rate so wrongTrailerCount stays >0 (has margin ~9).

## F-08 ‚Äî one real web‚Üîserver e2e (no stubbed boundaries)

**Verified gap:** every `packages/web/test/*.e2e.ts` stubs `/api/*` (page.route) + ws (routeWebSocket).
No browser exercises the real web‚ÜíFastify‚Üíprojection‚ÜíPostgres chain. **Critical wiring fact:** web
talks same-origin `/api/*` + `/api/ws`; Fastify registers routes at ROOT (`/hubs`,`/ws`,...). Dev
vite `server.proxy` strips `/api`; **prod `vite preview` has NO proxy** ‚Üí a real e2e at :4173 would
404. RESOLUTION: vite7 `PreviewOptions` supports `proxy` (verified in index.d.ts) ‚Äî add `preview.proxy`.

**Approach ‚Äî ONE keystone real-path spec + minimal harness; reuse everything, isolate to a new
project/port so the fast hermetic suite is untouched.** Real API booted ONCE in Playwright
`globalSetup` (programmatic `buildServer` + synchronous `driveSimulation` against `startPgFixture`
testcontainers PG ‚Äî deterministic, no race vs paced stream), torn down in `globalTeardown`.

**Change plan:**
1. NEW `packages/web/test/real-e2e.globalSetup.ts` ‚Äî `fx=await startPgFixture(); built=await buildServer({db:fx.db,enableWs:true,simSeed:4242,baselineTicks:120}); await driveSimulation({db:fx.db,seed:4242,durationTicks:120,rfid:DEMO_RFID_CONFIG,broadcast:built.broadcast,loop:built.loop}); await built.app.listen({port:MM_E2E_API_PORT??3101,host:'127.0.0.1'})`; stash {fx,built} for teardown. Long timeout (cold testcontainer pull).
2. NEW `packages/web/test/real-e2e.globalTeardown.ts` ‚Äî close app + stop fixture.
3. NEW `packages/web/vite.preview-real.config.ts` ‚Äî `preview:{port:4273, proxy:{'/api':{target:'http://localhost:'+(MM_E2E_API_PORT??3101), changeOrigin:true, ws:true, rewrite:p=>p.replace(/^\/api/,'')}}}`. Reuse react() plugin; same build output (reuse `pnpm build` artifacts).
4. `packages/web/package.json` ‚Äî `"preview:real":"vite preview --config vite.preview-real.config.ts --port 4273"`, `"test:e2e:real":"playwright test --project=chromium-real"`.
5. `packages/web/playwright.config.ts` ‚Äî `globalSetup`/`globalTeardown`; new project `{name:'chromium-real', testMatch:/real-e2e\.e2e\.ts$/, use:{...Desktop Chrome, baseURL:'http://localhost:4273'}}`; `testIgnore:/real-e2e\.e2e\.ts$/` on existing 'chromium'; webServer entry `{command:'pnpm build && pnpm preview:real', url:'http://localhost:4273', reuseExistingServer:!CI, timeout:180_000}`. **Gate the heavy globalSetup boot** behind project/env (e.g. only when `MM_E2E_REAL=1` / `--project=chromium-real`) so the fast suite doesn't need Docker.
6. NEW `packages/web/test/real-e2e.e2e.ts` ‚Äî NO stubs. (A) map renders real hubs+routes (cross-check data-hub-count vs a direct `GET /hubs`); (B) KPI panel `kpi-value-wrongTrailerCount` polls >0; (C) alert feed has an entry `[data-kind="wrongTrailer"]` with text "Wrong Trailer" ‚Äî **the F-01 catcher**. Generous `expect.poll` timeouts (ws snapshot arrives post-connect).
7. `packages/api/test/pg-fixture.ts` ‚Äî ensure importable from web (relative import or a tiny test-utils export).
8. root `package.json`/CI ‚Äî `"test:e2e:real":"turbo run build --filter @mm/api --filter @mm/web && pnpm --filter @mm/web test:e2e:real"`; separate CI job gated on Docker.

**TDD:** (1) RED: write the spec + run BEFORE adding proxy ‚Üí must fail Test 1 with /api/hubs 404 (proves it hits the live boundary). (2) Add proxy+setup+project ‚Üí GREEN. (3) Contract-regression proof: temporarily break the prefix or exceptionKindToWire ‚Üí relevant test goes RED (demonstrates it catches F-01). Revert. (4) Determinism: run twice. (5) Existing hermetic projects stay green + don't pick up the real spec. Build first.

**Risks:** Docker/OrbStack availability (honor MM_PG_URL override; separate CI job; 180s timeouts for cold pull). Cross-package pg-fixture import resolution. ws-through-proxy timing ‚Üí expect.poll, not immediate. Port collisions (3101/4273). Single global-setup constraint ‚Üí gate the boot behind project/env.
