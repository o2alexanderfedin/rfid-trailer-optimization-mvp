# Requirements — Milestone v1.1 "Realistic Time Model + Hardening"

**Defined:** 2026-06-21
**Milestone goal:** Make the simulated time model defensible end-to-end — so the rolling optimizer plans against realistic, geography-derived dwell+transit instead of flat constants — and close the post-v1.0 audit follow-ups.
**Continues REQ-ID numbering from v1.0** (OPT-01..08, VIZ-01..05, etc.). v1.0's 48 requirements are archived in `milestones/v1.0-REQUIREMENTS.md`.

## v1.1 Requirements

### Optimizer time-awareness (OPT)
- [x] **OPT-09**: The rolling optimizer plans against **expected dwell + transit** derived from the timing model — folding them into the time-expanded graph's trip travel (`OptimizerRoute.travelMin`) and a hub **service-time** offset — instead of fixed 15-min steps with flat dwell. A change to the timing config produces a correspondingly different plan.
  - **Outcome:** `TwinRoute.travelMin` feeds both the time-expanded MCF graph and VRPTW oracle; VRPTW `RouteStop.serviceMin` carries role-based expected dwell. Changing timing config changes the plan (verified at pure-core boundary). 32 OPT-09/10 unit tests green.
- [x] **OPT-10**: The optimizer's per-leg/per-hub planning estimate is derived **deterministically** from the timing distribution config via a single shared estimator (`expectedMinutes` = clamped `median·exp(σ²/2)`, the distribution **mean**), so planner output is reproducible and the same config drives both the simulator's random draw and the planner's estimate (DRY).
  - **Outcome:** Single `expectedMinutes` in `@mm/domain` (`timing.ts`); consumed by both simulator and optimizer. `timing.test.ts` asserts analytic mean values, mean ≥ median for σ>0, clamp behavior. All pass.

### Realistic time model (TIME)
- [x] **TIME-01**: Per-leg transit **medians are derived from real route distance/geography** (precomputed road distance ÷ representative HGV speed), so each leg has a distance-appropriate transit distribution rather than a single flat ~30-min median.
  - **Outcome:** Initially haversine @ 80 km/h; upgraded to ORS `duration_s` for all 18 legs (committed `road-geometry.generated.json`). 18 distinct medians; default stream differs byte-for-byte from flat-override stream.
- [x] **TIME-02**: A distinct **center-hub re-dispatch dwell** is modeled (cross-dock unload → re-sort → re-dispatch) so the wired-but-unused `dwellCenter` distribution applies at the center, separate from spoke dwell — with exactly one dwell applied per stop (no double-count).
  - **Outcome:** `engine.ts arriveTrailer` draws `spokeDwell` and `centerDwell` unconditionally per turnaround, role-keyed. Empirically confirmed: 58 turnaround gaps avg 95.8 ticks (spoke ~27 + center ~65). `center-dwell.unit.test.ts` asserts exact gap = SPOKE+CENTER with no double-count.

### Visualization (VIZ)
- [x] **VIZ-06**: Route geometry **follows real roads** — trailers animate along precomputed OpenRouteService `driving-hgv` polylines (committed as static GeoJSON to preserve determinism) instead of straight great-circle arcs; downstream ws/animation is unchanged (geometry stays `[lon,lat][]`).
  - **Outcome:** Committed `road-geometry.generated.json` (RDP-simplified 8.5 MB → 82 KB, ~72 pts/leg, hub-checksum drift guard). Road-following verified (0.71° deviation, 20–175 vertices/leg vs 24-pt great-circle). 13/13 sim-unit + twin-snapshot tests pass.

### Hardening (HRD)
- [x] **HRD-01**: `parseEnvelope` **tolerates a missing `speed` field**, falling back to a `DEFAULT_SPEED` (with a one-time warning) so a partial/older server envelope still animates the map — while still rejecting genuinely malformed envelopes (bad `v`/`type`/`seq`/`simMs`/`payload`).
  - **Outcome:** Fallback built into `parseEnvelope` in `wsClient.ts`; `console.warn` fires exactly once (module-level guard); malformed core fields still rejected to `null`. `isSimSpeedState` upgraded to a real type guard. Tests assert all branches incl. warn-exactly-once.

### Quality (QA)
- [x] **QA-01**: Coverage top-up — the `wsClient` socket path (`useWsEnvelope`: open-once, seq-gap→resync, snapshot-replace, tick-apply) and overall **branch** coverage are raised toward the project bar with **behavior-asserting** tests (no metric gaming).
  - **Outcome:** Four named socket behaviors tested via real MSW WebSocket through production `WsProvider`. seq-gap asserts exact outbound `{v:1,type:"resync"}` frame. `wsClient.ts` parse path at 100%. Orphaned `useWsEnvelope` dead-code removed (`18c68ce`).

## Out of Scope (v1.1)

- **Full stochastic / robust optimization** (scenario sampling, Monte-Carlo planning, chance constraints) — v1.1 uses a single deterministic expected-value estimate.
- **Service-level / percentile planning estimate** — the **mean** is the v1.1 estimator.
- **Runtime / live routing** (OSRM/Valhalla self-host, live ORS calls, time-of-day traffic) — road data is precomputed offline and committed.
- **HGV hours-of-service / legally-mandated break modeling** — average overhead absorbed into medians.
- Carried-forward v1.0 deferrals (live 8-metric A/B money slide, durable idempotency, ws-connection consolidation, true-volume utilization).
