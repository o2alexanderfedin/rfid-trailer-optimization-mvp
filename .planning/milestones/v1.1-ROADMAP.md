# Milestone v1.1: Realistic Time Model + Hardening

**Status:** ✅ SHIPPED 2026-06-22
**Phases:** 6–8
**Total Plans:** 3 phases (no sub-plans; each phase executed as a single integrated work unit)

## Overview

Makes the simulated time model defensible end-to-end — so the rolling optimizer plans against
realistic, geography-derived dwell+transit instead of flat constants — and closes the
post-v1.0 audit follow-ups. Cross-cutting: determinism mandatory, TDD, strict TS (no `any`).

Key deliverables: shared `expectedMinutes` mean estimator in `@mm/domain`; center re-dispatch
dwell (`dwellCenter`); real ORS `driving-hgv` road polylines (RDP-simplified 8.5 MB→82 KB);
ORS-duration transit medians end-to-end; optimizer plans against expected timing (OPT-09/10);
tolerant `parseEnvelope` speed fallback (HRD-01); ws socket-path coverage (QA-01).

### Key Decisions

- **Deterministic planning estimate = log-normal MEAN** (`median·exp(σ²/2)`), not median (optimistic) or percentile (over-conservative) — single shared `expectedMinutes` in `@mm/domain` keeps simulator draw + planner estimate DRY (OPT-10).
- **Transit scale = realistic-absolute** (ORS `duration_s` for all 18 legs; haversine fallback) — demo watchability handled by the 120× sim-speed gauge; lifecycle integration tests use the DIP flat-timing override.
- **Road geometry committed as static GeoJSON** (RDP-simplified 82 KB) with a hub-checksum drift guard — preserves determinism; no live ORS in the hot path.
- **Dwell modeled as VRPTW `serviceMin` only** (not an additional flow-graph offset) to avoid double-counting against existing `wait`/`hold` edges (PITFALLS P4).
- **Orphaned hook excised** — `useWsEnvelope` hook and its inflating test removed; coverage now reflects only the live `WsProvider` path.

### Issues Resolved

- Integration suite failures (12 failures / 6 files) after Phase 6 — lifecycle tests needed flat-timing DIP pins for realistic transit horizons.
- `tsc` does not emit data files — added `copy-assets` step so built `@mm/api` resolves `road-geometry.generated.json`.
- 3 stray NUL bytes in `epoch.ts` (used as Map-key separators, behavior-correct but blocked grep/Edit) — replaced with spaces.
- Orphaned `useWsEnvelope` hook (dead code) inflating coverage — removed in `18c68ce`.

### Issues Deferred

- Optimizer `RollingLoop`/`buildTwinSnapshot`/`runEpoch` hardcode `DEFAULT_TIMING_CONFIG` on the live path (asymmetric plumbing — harmless as shipped; latent DRY break if non-default timing ever injected).
- "Config change → different plan" proven only at the pure-core boundary (not end-to-end through the live server).
- `GET /routes` API integration assertion only checks `length > 1` (not road-vs-great-circle distinction at API boundary).

### Technical Debt (LOW — deferred to v1.2)

1. **Timing-config plumbing asymmetric (optimizer):** `RollingLoop`/`buildTwinSnapshot`/`runEpoch` hardcode `DEFAULT_TIMING_CONFIG`; `deps.timing` is forwarded to the simulator but not threaded to the optimizer. Production default agrees on both sides — harmless as shipped. Recommend threading `deps.timing` through `RollingLoop`/`buildTwinSnapshot` or documenting demo as fixed to default.
2. **`DEFAULT_SPEED` literal duplicated** in `wsClient.ts`, `SpeedControl.tsx`, and the backend speed-controller — could drift; a shared constant is a future DRY cleanup.
3. **`seq`/`simMs` accept `NaN`** (`typeof === "number"`) — pre-existing, outside HRD-01's surface.
4. **`GET /routes` integration assertion weak** — only checks `geometry.length > 1`; a great-circle fallback would also pass; recommend tightening to a known multi-vertex leg.
