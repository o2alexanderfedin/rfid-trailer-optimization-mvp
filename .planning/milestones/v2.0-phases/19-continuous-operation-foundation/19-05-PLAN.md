<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WS client → server (WsEnvelope) | simDay is server-computed from virtual clock; clients cannot inject it |
| sendRawIfOpen → WS buffer | backpressure guard reads socket.bufferedAmount (OS-managed); cannot be manipulated by application layer |

## STRIDE Threat Register

No new authentication, cryptography, or external-input attack surface. simDay is a server-derived integer. The backpressure guard is a mitigation for resource exhaustion, not a security control.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-05 | Denial of Service | WS broadcast to backgrounded clients | mitigate | bufferedAmount > 256KB guard in sendRawIfOpen prevents server-side buffer saturation from backgrounded tabs. Initial snapshot is unguarded (correct: fresh connect needs the full state). |
| T-19-05b | Information Disclosure | simDay in WsEnvelope | accept | simDay is virtual simulation time (not real clock), carries no PII or sensitive data. Acceptable to broadcast to all connected ws clients. |
</threat_model>
