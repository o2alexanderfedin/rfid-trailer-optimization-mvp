## Problem
`driveSimulationOpenEnded` calls `generateChunk(horizonTick)` = `simulate({durationTicks: horizonTick})`, **regenerating the whole event prefix from tick 0 each chunk**: O(n) memory (full `ticks`/`destHub` held), O(n²) compute. And the Postgres **event log + projections grow unboundedly** over a continuous run. Neither can sustain genuinely indefinite operation.

## Task A — Resumable engine (continuation), `@mm/simulation`
1. Introduce an explicit `SimContinuation` value (NOT a JS generator): carries everything needed to resume deterministically — `nextTick` (virtual time), the EventQueue contents, all seeded RNG sub-stream states (raw), entity/world state (`pendingBySpoke`, package registry, counters), and the monotonic `nextSequenceId`. Plain serializable data; deterministic field order.
2. Core engine becomes continuation-driven: a function that, given `SimContinuation | freshSeed` and a target `horizonTick`, runs the EventQueue up to `horizonTick` and returns `{ events, continuation }`.
3. **`simulate(opts)` (finite, all-at-once) MUST stay byte-identical** — reimplement it as: start fresh → run continuation to `durationTicks` → collect all events. The existing seed-1234 golden and seed-42 10k golden must not change.
4. Keep the deterministic same-tick tie-break `(occurredAt/tick, sequenceId)`. Keep sim RNG exclusive (existing salts). No wall-clock in the engine (uses `occurredAt`/virtual already).

## Task B — Driver uses continuation, bounded working set, `@mm/api`
1. `driveSimulationOpenEnded` holds ONE `SimContinuation`. Each step: pull the next `chunkTicks` window via the continuation (no regen), emit/drain those events, then **discard drained ticks** and prune `destHub` for processed packages → bounded RAM (window-sized, not run-sized).
2. Remove the growing `ticks = generateChunk(growingHorizon)` regen. The continuation IS the state; the prefix is never recomputed.

## Task C — Bounded persisted retention (LIVE CONTINUOUS PATH ONLY), `@mm/api`
1. **Event-log retention:** periodically `DELETE FROM events WHERE global_seq <= (watermark - RETENTION_MARGIN)`. NEVER delete at/above the projection watermark (those aren't yet projected). Safe because catch-up resumes from the watermark (projections are the snapshot) — pruned events are already materialized and never replayed at runtime. Gated on the continuous/opt-in path; configurable margin (sensible default).
2. **Projection retention:** purge package rows (`package_location`/`hub_inventory`/`zone_estimate`) for packages inactive beyond a retention horizon (stale), bounding projection tables in a delivery-less Phase-19 run. Design it generic so Phase 22's `PackageDelivered` purge composes with it.
3. **Finite/test path: retention OFF** → full log retained → replay-from-0 byte-identical. Add a guard/test asserting finite/test paths never read a pruned/live log.
