<objective>
Build the load-bearing correctness core: the **greedy route-aware planner** (LOAD-03), the
**INDEPENDENT feasibility validator** (LOAD-04 — a separate code path that recomputes blockers from
the placed slices via "virtual unload simulation", NOT from the planner's bookkeeping), and
**partial-LIFO** acceptance of bounded blockers (LOAD-05). Then the two keystone tests that make
silent LIFO lies impossible: the **golden deliberately-reversed-plan fixture** (the single most
important test in the codebase — it MUST be flagged HARD-infeasible) and the **planner-vs-validator
property test** (fuzzed inputs: planner output satisfies the canonical invariant AND the validator
agrees on the planner's feasibility classification).

Purpose: this is the anti-P1/anti-P2 heart of the phase. The validator must be an independent recompute
(it walks the route stop-by-stop over the placed slice contents and counts blockers) so it cannot share
the planner's bug. Feasibility (`{hardViolations, softViolations}`) is a HARD GATE kept entirely
separate from any score (P2). After this slice, a real user gets a plan plus an honest feasible/infeasible
verdict that an independent path verified.

Output: `plan-load.ts`, `validator.ts`, the golden fixture, and the property test — all in the existing
`@mm/load-planner` package (foundation from Plan 03).
</objective>
