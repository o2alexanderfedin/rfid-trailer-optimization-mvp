<objective>
Create `@mm/aggregation` (AGG-01..04): a PURE, IO-free package that groups `PlanningPackage[]` into
`LoadBlock[]` by the 7-part key, computes aggregate volume/weight/count, splits oversized or
handling-incompatible blocks into feasible sub-blocks, and assigns each block an SLA+deadline-derived
priority. This is the first vertical slice of the Phase-2 user story — after it lands, a real user can
hand the system raw packages and get back scored, prioritized, feasibly-sized load blocks.

Purpose: aggregation is Layer 1 of the planning pipeline and the input to the load planner. It must be
deterministic (stable grouping/splitting iteration order) so the whole phase replays identically
(P3 discipline carried from Phase 1). Keeping it pure mirrors `@mm/projections` reducers and makes it
trivially TDD-able — the roadmap's explicit ask.

Output: a new `packages/aggregation` package importing ONLY `@mm/domain`, with `aggregate`,
`splitBlock`, `blockPriority`, and `deadlineBucket`, fully unit-tested RED→GREEN→REFACTOR.
</objective>
