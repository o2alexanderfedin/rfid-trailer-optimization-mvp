<objective>
Extend the deterministic `@mm/simulation` engine (SIM-03) to emit probabilistic `RfidObserved` events:
at dock-door portals (when a package is docked/loaded) and at trailer antennas (during dwell), with a
configurable `missRate` (some reads dropped) and `rssiNoise` (RSSI jitter, occasional wrong-zone/wrong-tag
reads). Reads must be seeded and deterministic — same seed ⇒ identical RFID stream, drops and noise included.

Purpose: the sim is the ONLY data source for Phase 3. Probabilistic reads (not perfect ones) are what make
the fusion + detection demo credible — they exercise the confidence model, the dwell windowing (bursts of
reads per dwell), and the anti-P6 "missing read ≠ missing package" guarantee. No real hardware.

Output: the simulator now interleaves RfidObserved into its existing deterministic stream; packages carry
an rfidTagId; the RFID stream is reproducible per seed.
</objective>

<interfaces>
<!-- The deterministic engine internals to extend. ALL randomness MUST flow through `Rng`; ALL time through the clock. -->

The Rng surface (rng.ts) — the ONLY randomness source (no Math.random anywhere):
```typescript
interface Rng { next: () => number; int: (maxExclusive: number) => number; pick: <T>(items: readonly T[]) => T; }
```

The generation core (engine.ts) emits via `emit(streamId, event)` at the current `clock.nowIso()`:
  - packages are created in `createPackageBatch` (add an rfidTagId here, deterministic from the counter)
  - trailers depart in `departTrailer` (load scans happen here — PORTAL read opportunity)
  - trailers arrive/dock/unload in `arriveTrailer` (dock + unload scans — PORTAL + ANTENNA read opportunity)
  - the event-queue is a `(fireTick, seq)` stable order; ties break deterministically.

The RfidObserved event (from @mm/domain, Plan 01):
  payload: { tagId, readerId, antennaId, rssi, trailerId, hubId, confidence }
  (confidence here is the SIMULATOR's per-read confidence; the fusion engine recomputes its own from rssi.)

SimulatedEvent shape (already exported): { streamId, event, occurredAt }.
SimulateOptions: { seed, durationTicks } — EXTEND with optional RFID knobs (defaulted so existing tests stay green).

<feature>
  <name>Seeded probabilistic RFID emission in the simulator (SIM-03)</name>
  <files>packages/simulation/src/rfid.ts, packages/simulation/src/engine.ts, packages/simulation/src/index.ts</files>
  <behavior>
    - Every package created gets a deterministic `rfidTagId` (e.g. `TAG-${packageId}`), set on the
      PackageCreated payload (now possible per Plan 01's optional rfidTagId).
    - During load (departTrailer) a DOCK-PORTAL reader emits RfidObserved for each loaded package's tag,
      with a strong RSSI (portal = high reliability), subject to missRate.
    - During dwell/unload (arriveTrailer) a TRAILER-ANTENNA reader emits a BURST of reads per carried
      package's tag (zone-ish, lower/noisier RSSI), subject to missRate — multiple reads per dwell so the
      fusion engine's dwell windowing is exercised.
    - `missRate` ∈ [0,1]: each candidate read is dropped with probability missRate (via rng). missRate=0
      ⇒ all reads emitted; missRate near 1 ⇒ most dropped.
    - `rssiNoise`: RSSI is the reader-type base ± a seeded jitter; an occasional (small-rate) wrong-zone or
      wrong-tag read can be injected, all via rng.
    - DETERMINISM: identical seed ⇒ byte-identical event stream (RFID reads, drops, and noise included),
      because every miss/noise/jitter decision uses the seeded Rng and the existing stable event queue.
    - The simulator NEVER emits a "package missing" signal — a dropped read is simply an omitted event
      (this is the data-level half of the anti-P6 guarantee; detection enforces the other half in Plan 06).
  </behavior>
  <implementation>
    Put RFID generation in a new `rfid.ts` (`emitRfidReads(...)` + `RfidSimConfig` with defaults) so the
    engine stays readable. Thread an `Rng` and the emit callback in (dependency injection — keep rfid.ts
    free of ambient state). Extend `SimulateOptions`/`RunSimulationOptions` with optional
    `rfid?: Partial<RfidSimConfig>` (default: a modest missRate + rssiNoise) so the existing Phase-1/2 tests
    that pass only `{ seed, durationTicks }` still produce their original non-RFID stream PLUS RFID reads —
    OR gate RFID behind the presence of the rfid option if any existing determinism golden fixture would
    break. CHOOSE the approach that keeps existing simulation golden tests green; document the choice in the
    SUMMARY. Reader/antenna ids are derived deterministically from hub/trailer ids (e.g. `${hubId}-PORTAL`,
    `${trailerId}-ANT`).

    TDD: write rfid.unit.test.ts + rfid-determinism.unit.test.ts FIRST (RED), then implement. Commits:
      - test(03-03): failing RFID emission + determinism tests
      - feat(03-03): emit seeded probabilistic RFID reads
  </implementation>
</feature>
