The schema factory (events/schemas.ts) — every event is `{ type, schemaVersion, payload }`:
```typescript
const id = z.string().min(1);

const schemaVersion = z.literal(PHASE1_SCHEMA_VERSION);
function eventSchema<TType extends string, TShape extends z.ZodRawShape>(
  type: TType, payload: z.ZodObject<TShape>,
): z.ZodObject<{ type: z.ZodLiteral<TType>; schemaVersion: ...; payload: ... }>;
export const domainEventSchema = z.discriminatedUnion("type", [ /* 8 members today */ ]);
```

The union (events/domain-event.ts) — each member is `z.infer<typeof xSchema>`:
```typescript
export type DomainEvent = HubRegistered | RouteRegistered | PackageCreated
  | PackageScanned | PackageArrivedAtHub | TrailerDeparted | TrailerArrivedAtHub | TrailerDocked;
export type DomainEventType = DomainEvent["type"];
export function assertNever(value: never): never; // throws
```

The build gate (events/contract.assert.ts) — these two constructs FAIL THE BUILD if incomplete:
  1. `assertExhaustive(event: DomainEvent)` — a switch with a `case` per `event.type`, `default: assertNever(event)`.
  2. `const _zodMatchesHandWrittenUnion: Exact<z.infer<typeof domainEventSchema>, DomainEvent> = true;`

The current PackageCreated payload (events/schemas.ts) — `rfidTagId` must be ADDED here (optional):
```typescript
z.object({ packageId, originHubId, destHubId, sizeClass, weight: z.number().positive() })
```
NOTE: payloads are `.strict()` (the factory calls `payload.strict()`), so a NEW optional field MUST be
declared in the schema or it is rejected. Add `rfidTagId: id.optional()`.
