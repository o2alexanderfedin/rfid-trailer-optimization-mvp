<objective>
Replace the single legacy ws snapshot message with the versioned keyframe+delta envelope (VIZ-04) that every Phase-5 frontend consumes. This is the protocol contract the animation, coloring, exception feed, KPI dashboard, and plan-delta panels all read — so it lands FIRST, in Wave 1, before any frontend plan.

Per 05-RESEARCH.md Q3: a `snapshot` (full baseline, on connect/resync) + a per-tick `tick` (only changed fields keyed by entity id), each stamped with `seq` (drop detection) and `simMs` (authoritative sim clock for client resync). One message per sim tick — NEVER one per raw domain event (ARCHITECTURE Anti-Pattern 4 / threat T-01-19).

Purpose: every downstream Phase-5 frontend tween, color bucket, feed entry, and KPI partial flows through this envelope; getting it right (versioned, seq+simMs, upsert/delete keyed by id) is load-bearing for the whole wrapper.
Output: `packages/api/src/ws/envelope.ts` (typed union + builders) and a rewritten `snapshots.ts` that emits it.
</objective>
