  <name>Weighted objective + feasibility hard-gate selection + local repair (OPT-07/08)</name>
  <files>packages/optimizer/src/objective/objective.ts, packages/optimizer/src/objective/select-plan.ts, packages/optimizer/src/repair/local-repair.ts</files>
  <behavior>
    objective:
    - A metrics fixture yields the exact hand-computed weighted sum (every term contributes per its weight).
    - objective takes NO feasibility argument (compile-enforced + test: passing a FeasibilityResult is a type error / not accepted).
    KEYSTONE feasibility-hard-gate (anti-P2):
    - Given a low-objective-but-INFEASIBLE candidate and a higher-objective-but-FEASIBLE candidate, selectPlan returns the FEASIBLE one. The infeasible candidate is rejected regardless of its lower score.
    - objective(metrics) and feasibility are observed as two separate values on the Candidate; no code path collapses them.
    selectPlan:
    - Among feasible candidates picks the minimum objective; ties broken by planId lexicographically (deterministic, anti-P7).
    - Returns null when no candidate is feasible.
    localRepair:
    - An infeasible plan yields >= 1 feasible recommendation among split/reassign/hold/over-carry; each carries a non-empty human-readable rationale string.
    - The first returned recommendation is the best feasible by objective (ranked).
    - Determinism: same input ⇒ same ranked recommendations.
  </behavior>
  <implementation>
    GREEN: implement `objective(metrics, weights)` as a pure weighted sum of all §12 terms (rehandle + utilization terms sourced from the REUSED scorePlan; churn from churnVsPrevious for anti-P7). `selectPlan` filters to `isFeasible(c.feasibility)` FIRST, then picks min objective with planId tie-break, returns null if none feasible — feasibility is NEVER added into the objective number. `localRepair` generates candidate variants (split a block, reassign to another trailer/leg, hold to a later epoch, over-carry past a hub), validates each via the REUSED validatePlan, scores feasible ones via objective, attaches a rationale per §17.4, returns ranked best-first. All pure, deterministic, integer-friendly; no Date.now/Math.random.
