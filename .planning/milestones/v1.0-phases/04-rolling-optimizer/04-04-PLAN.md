  <name>VRPTW construction + local search + routeTrailers (OPT-03)</name>
  <files>packages/optimizer/src/vrptw/construct.ts, packages/optimizer/src/vrptw/local-search.ts, packages/optimizer/src/vrptw/route-trailers.ts</files>
  <behavior>
    construction:
    - A 3-stop fixture with feasible windows produces a sequence visiting all stops within their windows; arrival within [windowStart, windowEnd] at each.
    - An insertion that would arrive after a stop's windowEnd is rejected (not added); a route that cannot fit all stops returns feasible:false (or splits per design).
    - Capacity: total demand on a route never exceeds capacity.
    local search:
    - Applying 2-opt/or-opt to a feasible route never increases total travel cost and never introduces a window/capacity violation (property test over seeded fixtures).
    - A window-violating candidate move is rejected.
    routeTrailers:
    - Returns RoutedStop ETAs (arrival/departure) consistent with TravelModel + service times, plus a utilization estimate (demand/capacity).
    - The resulting trailer load passes `validatePlan(...)` HARD gate via `isFeasible` (reused), proven on a fixture; a deliberately un-unloadable load fails the gate.
    determinism:
    - Same input ⇒ identical route + ETAs + utilization.
  </behavior>
  <implementation>
    GREEN: implement `constructRoutes` (cheapest-insertion: greedily insert each stop at the position minimizing added cost while keeping all windows + capacity feasible). Implement `localSearch` (2-opt + or-opt moves; accept a move only if it is feasible AND reduces cost). `routeTrailers` runs construct → localSearch, computes arrival/departure ETAs from the TravelModel + service times, computes utilization, and runs the trailer load through `@mm/load-planner`'s `validatePlan`/`isFeasible` to set `feasible`. All deterministic, integer-minute arithmetic, lexicographic tie-breaks, no RNG/clock.
    REFACTOR: factor window-feasibility + cost-delta checks into shared pure predicates used by both construct and local-search.
  </implementation>
