# Quick Task 260621-0fy — UI speed-of-time gauge

## Goal
A slider gauge (0.25×–8× of the default 120× sim speed) with a live `sim-min/real-sec`
readout and a Pause button, that actually changes how fast trucks move — by retuning the
**backend tick interval** live. Along the way, fix the latent frontend clock bug
(`simSpeed: 1` can't track the server cadence) so the trailer tween is smooth and synced.

## Background (verified from code)
- `simulation/engine.ts:87` `MS_PER_TICK = 60_000` (1 tick = 1 sim-minute).
- `api/sim/driver.ts:390,483` `driveSimulationPaced` captures `intervalMs = opts.tickIntervalMs ?? 500` ONCE, `setTimeout(intervalMs)` between ticks → 120× compression at the default.
- `web/map/MapView.tsx:88` `makeSimClock({ simSpeed: 1 })`; `simClock.ts` nudge-clamps each resync to `maxNudgeMs = 500`. By the math a simSpeed=1 clock gains ~1000 sim-ms/tick while the server jumps 60000 → it cannot track 120×.

## Speed model (single source of truth)
- Base: `tickIntervalMs = 500` ⇒ **1×** ⇒ 2 sim-min/real-sec ⇒ 120× compression.
- Multiplier `m` (relative to default): `tickIntervalMs = round(500 / m)`.
- Range `m ∈ [0.25, 8]` ⇒ interval ∈ `[62, 2000]` ms (clamped; never 0).
- Display readout: `sim-min/real-sec = 1000 / tickIntervalMs` (3× → ~6, matches the chosen UX).
- Frontend clock: `simSpeed (sim-ms/wall-ms) = MS_PER_TICK / tickIntervalMs = 60000 / tickIntervalMs` (= 120 at default); **0 while paused** so the tween freezes.
- Pause: a boolean; the driver holds before the next tick; the envelope reports `simSpeed: 0`.

## Invariants to preserve (regression guards)
- Determinism of the sim STREAM: interval/pause are pacing only (driver doc DETERMINISM CONTRACT) — never fed to `simulate`/event store/optimizer.
- OL leak + zero-alloc discipline: the gauge must NOT trigger map re-renders; map stays in refs; no new allocations in the postrender loop. `setSpeed` only mutates a closure number.
- Hermetic tests stay green; TypeScript strict (no `any`); no schema/optimizer changes.
