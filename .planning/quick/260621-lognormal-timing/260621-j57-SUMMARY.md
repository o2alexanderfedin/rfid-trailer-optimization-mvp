# Quick Task 260621-j57 ŌĆö Seeded log-normal dwell + transit ŌĆö SUMMARY

## What was built

Replaced the fixed `DWELL_TICKS=10` / `TRANSIT_TICKS=30` constants in
`@mm/simulation` with per-event **seeded log-normal** draws, so trailer hub-dwell
and leg-transit vary realistically (right-skewed) while the simulator stays
deterministic and reproducible.

### New: `packages/simulation/src/timing.ts`
- `sampleLogNormal(rng, {median, sigma, min, max})` ŌĆö pure, deterministic,
  clamped BoxŌĆōMuller draw (MINUTES; 1 tick = 1 min). Draws `u1,u2 = rng.next()`,
  floors `u1` at `Number.MIN_VALUE` so `ln(u1)` is finite, computes
  `z = sqrt(-2┬Ęln u1)┬Ęcos(2ŽĆ u2)`, `value = median┬Ęexp(sigma┬Ęz)`, clamps to
  `[min,max]`.
- `TimingConfig` (DIP) + `DEFAULT_TIMING_CONFIG`:
  - `dwellSpoke`  { median: 25, sigma: 0.4, min: 10, max: 180 }
  - `dwellCenter` { median: 60, sigma: 0.4, min: 10, max: 180 }
  - `transit`     { median: 30, sigma: 0.3, min: 10, max: 120 }
- Exported from the package index (`sampleLogNormal`, `DEFAULT_TIMING_CONFIG`,
  `LogNormalParams`, `TimingConfig`).

### Engine wiring: `packages/simulation/src/engine.ts`
- **Dedicated timing RNG**: `makeRng((seed ^ 0x00007717) >>> 0)`, separate from
  `rng` (operational), `rfidRng` (0x5f1da7c3), and `overCarryRng` (0x3ca71d5f).
  Timing draws never perturb those substreams.
- `drawTransitTicks()` = `max(1, round(sampleLogNormal(timingRng, transit)))`,
  drawn **per departure** at BOTH sites: outbound centerŌåÆspoke and the over-carry
  return spokeŌåÆcenter.
- `drawDwellTicks(role)` drawn **per arrival** at the spoke dwell-then-redispatch
  site.
- New `SimulateOptions.timing?: TimingConfig` (DIP) lets tests override the
  distributions; defaults to `DEFAULT_TIMING_CONFIG`.

## Distinct CENTER-dwell site? ŌĆö NO

The modeled cycle dwells-then-redispatches ONLY at the spoke
(`nextDepart = arriveTick + drawDwellTicks("spoke")`). The over-carried return
arrival at the center (`arriveOverCarriedAtCenter`) is a **terminal unload**, not
a re-dispatch, so there is no distinct center-dwell scheduling site to wire. Per
the instructions I did NOT invent one. `dwellCenter` IS defined in the config
(and unit-proven to sit above spoke dwell) so a future distinct center
re-dispatch can use it without a config change.
