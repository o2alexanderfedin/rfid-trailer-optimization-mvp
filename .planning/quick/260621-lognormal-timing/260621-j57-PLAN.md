# Quick Task 260621-j57 — Seeded log-normal dwell + transit

## Goal
Replace fixed `DWELL_TICKS=10` / `TRANSIT_TICKS=30` with seeded **log-normal** draws so trailer
hub-dwell and leg-transit vary realistically (right-skewed), while keeping the simulator's
deterministic, reproducible event stream.

## Confirmed design (user)
- Distribution: **log-normal**. Scope: **dwell + transit**.
- `sampleLogNormal(rng, {median, sigma, min, max})`: Box–Muller (`z = sqrt(-2 ln u1) cos(2π u2)`,
  guard `u1>0`), `value = median * exp(sigma*z)`, clamp `[min,max]`. Returns minutes (1 tick = 1 min).
- **Dedicated timing RNG** derived from the sim seed (e.g. `makeRng(seed ^ 0x7717)`) so timing draws
  are orthogonal to the RFID/over-carry RNG stream (those existing draws unchanged).
- Params (injectable config, DIP): dwell spoke median 25 / center 60, sigma ~0.4, clamp [10,180];
  transit median 30, sigma ~0.3, clamp [10,120].

## Tasks (TDD)
1. **sampleLogNormal + config** (pure, @mm/simulation) + unit test: seeded determinism (same seed →
   same sequence), median/spread within tolerance over N draws, clamps at bounds, `u1=0` guard.
2. **Wire engine.ts**: construct the timing rng; draw `dwellTicks` per ARRIVAL by hub role (spoke vs
   center — read the full cycle to find both dwell points; if center dwell isn't a distinct site,
   note it), `transitTicks` per DEPARTURE (both :323 outbound + :422 return). `round()` to whole ticks.
3. **Re-baseline** the golden-replay / deterministic-stream tests that assumed fixed 10/30. Keep the
   determinism contract (same seed → identical stream).
4. **Engine variation test**: over a seeded run, dwell + transit VARY across trailers, stay within
   bounds, and are byte-identical for a fixed seed.

## Out of scope
- Optimizer coupling (time-expanded graph `serviceMin` / wait-edge weights) — separate v1.1 item.
- Distance-derived transit (this randomizes around a median; distance-derived is a later upgrade).
