## Cross-Phase Integration (code-grounded, current source)

**Chain A — Driver HOS end-to-end: WIRED.** domain closed event union → sim emits (gated
`hosEnabled`, demo default ON via `detection-config.ts:163` / `main.ts:86`) → `driverStatusReducer`
+ `driverAssignmentReducer` via `applyInline` → `GET /api/hubs/:id/detail` (registered
`server.ts:121`) → ws `HubState` driver buckets (`ws/snapshots.ts`) → UI `HubDetail.tsx` +
`map/dutyColoring.ts`.

**Chain B — Optimizer HOS hard enforcement: WIRED (was DARK, now CLOSED).** The GAP-1 root cause
(`twin-snapshot.ts` building `TwinDriver` without `hosClock`) is fixed:
`driver-status.ts:158` persists the full `HosClock` → `schema.sql:64` `hos_clock JSONB` →
`inline.ts:229` serializes → `twin-snapshot.ts:395-401` attaches `hosClock` to `TwinDriver` →
`epoch.ts:302-308` `driverHosContextFor` returns a real context → `route-trailers.ts:245-262`
`hosLegsFeasible` hard gate fires → `epoch.ts:530-550` `firstHosInfeasibleLeg` → `local-repair.ts`
`hosVariants` emits insertRest/relay on `EpochRecommendation`. The running loop reaches it:
`server.ts:140-144` `RollingLoop({buildSnapshot})` → `driver.ts:386-391` (`applyInline` folds
projections, THEN `loop.tick()`) → `runEpoch`. Proven live by
`packages/api/test/optimizer-hos-live.int.test.ts` (real Postgres testcontainer).

**INFO (not a gap):** optional `Stop.restMin` (`optimizer/src/vrptw/types.ts:55`) is never
populated on the live epoch path (`stopsForTrailer` omits it, `?? 0`). By design — HOS
infeasibility is detected via `applyDrivingLeg` segments and recovered via `localRepair`
recommendations, not by auto-folding rest into the route. Back-compatible unused-optional.

never wall-clock.
